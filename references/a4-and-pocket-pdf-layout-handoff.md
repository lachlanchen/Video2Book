# A4 And Pocket PDF Layout Handoff

Date: 2026-04-18

This note explains the shared PDF-layout work that now lives in `Video2Book`
and can be reused by host repos such as `LazyEarn`, `LazyLearn`, and
`leonardsusskind`.

It covers four related fixes:

- wrapped running headers for normal A4/letter course PDFs
- wrapped running headers for compact pocket PDFs
- chapter-opening typography that keeps the older book-like style
- figure-refresh and overflow-repair steps that keep published PDFs aligned
  with corrected course sources

## Why This Was Needed

The generated lecture-note books had three recurring layout failures:

- long chapter headers overflowed the page width in normal full-size PDFs
- narrow pocket layouts either clipped headers or solved the problem by making
  the type too small
- corrected figure assets in a source course could leave already-published
  pocket PDFs stale unless the compact variants were rebuilt explicitly

The fix is now split into deterministic shared layout logic plus optional
Codex-assisted repair passes.

## Shared A4 Layout Fix

Normal full-size course PDFs compile through:

- `subtitles2notes/templates/lecture_notes_common_preamble.tex`

Generated courses receive a local copy of that template as:

- `generated_course_notes/<scope>/<course>/common_preamble.tex`

The current shared preamble now does the following:

- loads `fancyhdr`
- reserves a larger `\headheight`
- wraps the running chapter header inside a real parbox instead of assuming a
  single line
- raises that box upward into the existing top white space
- keeps explicit clearance above the headrule
- preserves the older uppercase italic chapter-header style instead of falling
  back to tiny sans/bold text

The important technical rule is:

1. wrap inside a real box
2. raise the box upward
3. still report the box height honestly
4. subtract a clearance band before the headrule

That is what prevents two-line headers from colliding with the horizontal rule.

## Shared Pocket Layout Fix

Compact exports are generated by:

- `scripts/export_course_pocket_pdfs.sh`

That script applies pocket-specific layout transforms at export time. The
current behavior is:

- chapter-opening blocks stay large, uppercase, and italic
- chapter openings move upward into the available white space before any strong
  font shrinkage is attempted
- running headers use wrapped headboxes with explicit clearance above the rule
- pocket exports also get narrow-page line-flow tuning for prose and equations

This means host repos do not need to fork their base LaTeX just to get usable
compact PDFs.

## Dynamic Repair Layers

Two dynamic repair layers now sit on top of the deterministic layout fixes.

### 1. Overflow repair

Use:

- `scripts/fix_latex_project_overfulls.sh`
- `scripts/fix_course_pocket_overfulls.sh`
- `scripts/start_pocket_overflow_fix_tmux.sh`

These tools:

- compile the project
- map actionable overfulls back to source files when possible
- audit rendered PDF output
- call Codex to patch the offending TeX
- recompile and keep only improved states

Use this after the shared layout fix, not before it. The layout layer should
solve global header behavior once; the Codex layer should handle local content
problems such as long equations or awkward figure/text interactions.

### 2. Figure recheck

Use:

- `scripts/recheck_course_figures.py`
- `scripts/process_course_figure_rechecks_one_by_one.sh`
- `scripts/start_course_figure_recheck_tmux.sh`

These tools:

- inspect existing screenshot-like lecture figures
- compare them against local caption/context plus transcript windows
- probe candidate frames from the source video
- use Codex with images to decide whether to keep or replace the current frame

This is the right tool when a lecture-note figure is semantically wrong even
though the LaTeX compiles cleanly.

## Recommended Reuse Pattern For LazyEarn And LazyLearn

For any host repo already using `Video2Book`, treat the workflow as:

1. update the shared A4 preamble source in `Video2Book`
2. propagate that template into each generated course's `common_preamble.tex`
3. rebuild the full-size published course PDFs
4. export pocket variants
5. run overflow fixing only if local content still breaks narrow layouts
6. run figure recheck if lecture screenshots are semantically wrong or stale

## Minimal Commands

From the host repo root:

```bash
cp Video2Book/subtitles2notes/templates/lecture_notes_common_preamble.tex \
  generated_course_notes/<scope>/<course>/common_preamble.tex
```

Rebuild the full-size course book:

```bash
cd generated_course_notes/<scope>/<course>
mkdir -p build
pdflatex -interaction=nonstopmode -halt-on-error -file-line-error \
  -output-directory=build course.tex
pdflatex -interaction=nonstopmode -halt-on-error -file-line-error \
  -output-directory=build course.tex
cp build/course.pdf course.pdf
```

Then export compact variants through `Video2Book`:

```bash
cd /path/to/Video2Book
./scripts/export_course_pocket_pdfs.sh --host-root /path/to/host-repo
```

If a specific course still has narrow-page breakage:

```bash
./scripts/start_pocket_overflow_fix_tmux.sh \
  --course <scope>/<course> \
  --font-mode onepointtwo \
  --model gpt-5.4 \
  --reasoning high
```

If a figure is visually wrong:

```bash
python3 scripts/recheck_course_figures.py \
  --repo-root /path/to/host-repo \
  --course <scope>/<course> \
  --apply
```

## What To Preserve

When adapting this in another host repo, preserve these decisions:

- do not solve header overflow by blindly shrinking fonts
- use the top white space before reducing type size
- keep the older uppercase italic chapter-header feel for both A4 and pocket
  books when that matches the host repo's visual language
- use deterministic layout fixes for systemic bugs
- use Codex repair only for local content problems
- rebuild pocket exports after correcting source figures or captions

## When A Host Repo Should Diverge

It is reasonable for `LazyEarn` or `LazyLearn` to diverge on:

- font family
- exact size values
- page size preset
- publishing destination paths

It should not diverge on the core mechanism:

- wrapped header boxes with explicit headrule clearance
- upward use of existing white space before aggressive shrinkage
- separate source-of-truth rebuilds for normal and pocket outputs
